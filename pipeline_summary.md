# EMS Transport Classification — Pipeline Summary

## Goal

Binary classification: given an EMS encounter, predict whether the patient was **transported** (1) or **refused transport** (0), using patient vitals, demographics, and the dispatch chief complaint.

---

## 1. Data Generation — `fhir_parser.py`

All data is synthetic Synthea-style FHIR JSON, generated by `write_mock_fhir_directory()`.

**Per patient:** random age (birth year 1940–2005), sex, 1–3 emergency encounters. Transport label set stochastically (~70% transport, 30% refusal) via a custom FHIR extension on each Encounter.

**Chief complaint (dispatch):** drawn from a weighted pool biased by the outcome label, so the dispatch text carries real predictive signal:

| Label | High-probability complaints | Low-probability complaints |
|---|---|---|
| Transport (1) | Chest pain, Shortness of breath, Syncope | General weakness, Fall injury, Abdominal pain |
| Refusal (0) | General weakness, Fall injury, Abdominal pain | Chest pain, Shortness of breath, Syncope |

Each pool is 70/30 weighted, giving the model a realistic but noisy text signal.

**Vitals:** six observations per encounter via standard LOINC codes. Means differ by class; larger noise (higher σ) creates realistic distribution overlap:

| Vital | Transport mean | Refusal mean | Noise σ |
|---|---|---|---|
| Heart rate (bpm) | 90 | 70 | 18 |
| Systolic BP (mmHg) | 130 | 118 | 22 |
| Diastolic BP (mmHg) | 82 | 76 | 14 |
| Respiratory rate | 22 | 16 | 5 |
| SpO2 (%) | 92 | 98 | 4 |
| Temperature (°F) | 99.2 | 98.4 | 1.0 |

~15% of vitals are randomly absent per encounter to simulate field measurement gaps. The parser selects the *most recent* observation within a 24-hour window around encounter start; missing vitals are later imputed with the training-set median.

---

## 2. Feature Engineering — `preprocessing.py: engineer_features()`

Before scaling, three clinically meaningful derived features are computed from the raw vitals:

| Feature | Formula | Clinical meaning |
|---|---|---|
| `shock_index` | `heart_rate / systolic_bp` | > 1.0 indicates haemodynamic compromise |
| `pulse_pressure` | `systolic_bp − diastolic_bp` | Proxy for stroke volume |
| `map_bp` | `diastolic_bp + pulse_pressure / 3` | Mean arterial pressure |

NaN inputs (missing vitals) propagate to NaN derived features, which the downstream `SimpleImputer(strategy="median")` handles transparently. `engineer_features()` is called at the start of `_select_columns()`, so every preprocessing path gets these features automatically.

---

## 3. Preprocessing — `preprocessing.py`

Input: DataFrame with columns `age, sex, heart_rate, systolic_bp, diastolic_bp, respiratory_rate, spo2, temperature, chief_complaint, transport`.

**Split:** 70% train / 15% val / 15% test, stratified on the transport label. The preprocessor is fitted on the training split only.

Three parallel encoding streams:

| Stream | Columns | Steps | Output type |
|---|---|---|---|
| Numeric | age + 6 raw vitals + 3 derived (13 total) | `SimpleImputer(median)` → `StandardScaler` | float32 |
| Categorical | sex | `SimpleImputer(mode)` → `OneHotEncoder` | float32 (appended to numeric) |
| Text | chief_complaint | `LabelEncoder` (index 0 = `<unknown>`) | int64 (separate tensor) |

Each DataLoader batch yields `(x_tab, x_text, y)`:
- `x_tab`  — float32 `[B, input_dim]`  (scaled numeric + one-hot sex; `input_dim ≈ 13`)
- `x_text` — int64   `[B]`             (complaint category index; `vocab_size = 7`)
- `y`      — float32 `[B, 1]`          (transport label)

`PreprocessResult` exposes `input_dim`, `vocab_size`, `feature_names`, `preprocessor`, and `label_encoder` for downstream model construction.

---

## 4. Model Architectures — `models.py`

All six PyTorch models share the forward signature `model(x_tab, x_text) → logit [B, 1]`. Tabular-only models accept but ignore `x_text`.

| Model | Type | Architecture | Uses dispatch? |
|---|---|---|---|
| `LogisticRegression` | Linear | `Linear(input_dim, 1)` | No |
| `ShallowMLP` | 1-layer MLP | `Linear→ReLU→Dropout→Linear` | No |
| `MLPClassifier` | 2-layer MLP | `Linear→BN→ReLU→Drop → Linear→BN→ReLU→Drop → Linear` | No |
| `DeepResidualMLP` | Residual MLP | Project → ResBlock → ResBlock → head (skip connections) | No |
| **`EmsClassifier`** ⭐ | **Two-branch** | **Tabular branch + `nn.Embedding` (16-d) → concat → head** | **Yes** |
| `AttentionEmsClassifier` | Attention + embedding | Soft feature-attention over vitals + dispatch embedding → head | Yes |

### Primary model: `EmsClassifier`

```
x_tab  [B, 13] → Linear(13, 64) → BatchNorm1d → ReLU → Dropout(0.2) → [B, 64]  ─┐
x_text [B]     → Embedding(7, 16)                                     → [B, 16]  ─┤ cat → [B, 80]
                                                                                    └→ Linear(80,32) → BN → ReLU → Dropout → Linear(32,1) → logit
```

The `nn.Embedding` layer (padding_idx=0) learns a 16-dimensional representation for each dispatch category jointly with the rest of the network. This lets the model combine physiological severity (vitals) with call type (dispatch) in a single end-to-end trained architecture.

---

## 5. Training — `train.py`

- **Loss:** `BCEWithLogitsLoss` with `pos_weight` auto-computed from training labels (`n_neg / n_pos`) to correct for the ~70/30 class imbalance.
- **Optimizer:** Adam, lr=1e-3, weight_decay=1e-4.
- **LR schedule:** `CosineAnnealingLR(T_max=epochs, eta_min=1e-5)` — smoothly decays learning rate from 1e-3 to 1e-5.
- **Early stopping:** monitors val loss; saves best checkpoint when `val_loss` improves by > 1e-6; stops after `patience` (default 12) stagnant epochs.
- **Output:** history dict (`train_loss`, `val_loss`, `train_acc`, `val_acc` per epoch) + optional training-curve PNG.

---

## 6. Evaluation — `eval.py`

`evaluate_binary(model, test_loader, device, val_loader=val_loader)`:

1. `collect_predictions()` runs the model over the loader, returning sigmoid probabilities and ground-truth labels.
2. `find_best_threshold()` sweeps thresholds [0.05, 0.95] in steps of 0.01 on the val set and selects the one maximising F1 — correcting for class imbalance without fixing the threshold at 0.5.
3. Reports on the test set at the tuned threshold: accuracy, precision, recall, F1, ROC-AUC, confusion matrix `[[TN, FP], [FN, TP]]`.

---

## 7. Model Comparison — `compare_models.py`

The full comparison script trains and evaluates every model in one run:

```
python compare_models.py [--mock-patients N] [--epochs N] [--workdir PATH]
```

**PyTorch models** — trained with the full `train_with_early_stopping` loop.  
**sklearn models** — `RandomForest`, `GradientBoosting`, `SVM (RBF)` fitted on train+val combined; receive scaled tabular + label-encoded text index as features.  
**Ensemble** — soft-vote average of `EmsClassifier` + `GradientBoosting` + `SVM` probabilities; threshold tuned on val set.

## 8. Entry Point — `main.py`

```
python main.py [--data-dir PATH] [--mock-patients N] [--epochs N]
               [--patience N] [--batch-size N] [--workdir PATH]
               [--skip-experiments] [--quiet-train] [--no-plot]
```

Run order:
1. Generate mock FHIR data if none found under `--data-dir`.
2. Parse FHIR → DataFrame.
3. Preprocess (feature engineering + three encoding streams) → DataLoaders.
4. Train `EmsClassifier` with early stopping.
5. Evaluate on test set using val-tuned threshold.
6. Run feature ablations, model comparison, and noise robustness experiments (`experiments.py`).
7. Save all results to `--workdir` (default: `artifacts/`).

For the full multi-model comparison (DL + sklearn + ensemble), use `compare_models.py` instead.
